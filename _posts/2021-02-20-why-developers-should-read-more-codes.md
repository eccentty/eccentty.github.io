Since I've started on this developer journey, I've come across the prevalent advice to read more good codes for one to be a better developer. Nonetheless, I was skeptical as I still thought doing more should help one to become better at doing something than reading. 

A year has passed and I can say doing more doesn't help me to write bettter codes. Doing more allowed me to make more mistakes and learn from them but in a scattered way i.e. only learning the parts where things don't work. And I still write poorly designed code.

If we were to compare programming to learning a language, then coding would probably be equivalent to speaking and writing while reading codes would obviously be equiavlent to reading. And we know that reading good books help to improve the grammar and vocabularies that are essential in learning a language. So this sense, reading good codes should help to improve our code structures and any language specifics implementation.

So from now on, I'm going to dedicate myself to reading more codes and will share keep you posted in any knowledge I've gained from the codes. And I hope you read more good codes too.
